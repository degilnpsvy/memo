2.5.1 Naming Rule of Models

Models have a naming rule. The name includes the prefix “rlxvm_” + module name + interface abstraction layer + data width. For example, a module type named “rlxvm_mctl_dw_ocp_tl2_64” means that the module is memory controller from DesignWare, whose interface is OCP TL2, whose data width is 64-bit.

Supported module types are shown in Supported Components of Chapter 1.2.

Supported module types can be shown with the option “-m” or “–show_registered_modules”.

The name of module instance has no limitation.
