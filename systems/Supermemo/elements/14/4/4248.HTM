2.3 Accuracy VS Speed

There is a tradeoff between how fast RLX-VM can run and how accurate RLX-VM is. To get more accuracy, we have to tolerate a slower simulation speed. We can set some parameters to increase simulation speed dramatically with only a little loss of accuracy. The simulation speed may be very important to early software development. So we introduce the related parameters here individually. Detailed information can be found in the individual sections below.

 Parameter “GRANULARITY” of the module rlxvm_rlxsim_ocp_tl2

 

Define this to 10 can have 12.7% or so performance gain.

 Parameter “QUICK_MEM_RANGES” of the module rlxvm_rlxsim_ocp_tl2

 

Define the RAM/ROM region to this parameter pair.

 Parameter “CHECK_NET_DELAY” of the module rlxvm_net_gateway

 

Define this to 100, simulation runs 5 times faster.

 

· Disable EJTAG in the processor model

Define “EJTAG = NO” in the processor’s configure form. (Also define the number of hardware breakpoints to 0 in the form)

 Use rlxvm_simple_memory_ocp_tl2 instead of rlxvm_mctl_dw_ocp_tl2

 

With the recommended parameters above, RLX-VM can achieve the simulation speed in the MIPS range. Boot up a Linux with shell can be within 1 minute or so.
