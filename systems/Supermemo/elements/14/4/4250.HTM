2.5 Configuration File

There is a global configure file, which is the first configuration file RLX-VM will read. Other configuration files for individual modules are specified in this global configuration file. The default global configuration file is INSTALL_DIR/platform/rlxocp/rlxvm_config_user.lua. Users are free to create their own global configuration file to construct some specific system architecture. When creating configuration file, user should keep in mind that the sequence of the items for one module in the configuration file should be kept in the order as illustrated in the default one because of some dependencies.

The top level configuration file is written in Lua language. (www.lua.org) However, users of RLX-VM are not required to master Lua. Configuration can be created following the example configuration file easily. In case your model needs a complicated configuration format, refer to “Lua 5.1 Reference Manual”, “3 – The Application Program Interface” (http://www.lua.org/manual/5.1/manual.html#3) for more detail.

User can designate his own global configuration file through the option “-c”. RLX-VM will try to find the other configuration files for individual modules in that directory too.

Parameters for each model instance are included in a Lua table. There is always a table named “global”, whose parameter has effect to all model instances. There is also a table named “top”, which is for the generic engine.

There are three common parameters for all models. They are explained here and may not be mentioned inside the individual model.

l SHOW_TLM

 Type: bool

Description: Set this parameter to true to show OCP transaction logging.

l SHOW_ACTION

 Type: integer

Description: Set this parameter to a positive value to tell the module to output the internal action information. The larger the value is the more detail information is shown. The maximum value for this parameter is 10. No internal action is shown if this parameter is set to 0.

l CLOCK_PERIOD

Type: string

Description: This defines the clock period of the model, which is a string, and has the format as “40 NS”, namely frequency is 25MHz. The first part of the string is a double value, and the second part is a time unit, which can be “FS”, “PS”, “NS”, “US”, “MS”, “SEC”.

There are also command line options, which are related to SHOW_TLM and SHOW_ACTION. If any one of them turns on the logging, the information will be shown.
