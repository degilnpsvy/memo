1.3.2 vmenc, vmdec

These are the encryption and decryption tools. The configuration files end user receives can be in encrypted form. Refer to Chap.2.5.2 for detail.

Usage:

 encryption

 

vmenc –i filename –o filename –s mark –h

-i filename: file or directory name to be encrypted. If it’s a directory name (only work

when –o option is off), program will access all the files under this directory and its subdirectory.

-o filename: output file name. If this option is on, input name will only be access as a file’s name. If no filename is specified, “_enc” will be add to file name.

-s mark : input file name filter. If this option is on, only the file that contains mark will be accessed.

-h help : program help information.

Example:

vmenc –i . –s .h

Encrypt files with “.h” under current directory and its subdirectory.

vmenc –i config.h

Encrypt files config.h to config_enc.h.

 decryption

 

vmdec –i filename –o filename –s mark –h

-i filename: file or directory name to be decrypted. If it’s a directory name (only work when –o option is off), program will access all the files under this directory and its subdirectory.

-o filename: output file name. If this option is on, input name will only be access as a file’s name.

-s mark : input file name filter. If this option is on, only the file that contains mark will be accessed.

-h help : program help information.

Example:

vmdec –i . –s .h

Decrypt files with “.h” under current directory and its subdirectory.

vmdec –i Config_enc.h –o dec.h

Decrypt file Config_enc.h to file dec.h.
