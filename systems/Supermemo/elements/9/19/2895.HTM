6.4.2 Using ASIDs

By setting up TLB entries with a particular ASID setting and with the Entry Lo0-1(G) bit set 0, those entries will only ever be used when the CPU’s thencurrent EntryHi(ASID) register field matches the TLB entry’s value. Since EntryHi (ASID) is an 8-bit field, that allows you to map up to 256 different address spaces simultaneously, without requiring that you clear out the TLB on a context change. If you do run out of ASIDs, you will have to pick some process that you can completely throw out of the TLB. Once you’ve discarded all its mappings, you can revoke its ASID value and reapply to some other process.
