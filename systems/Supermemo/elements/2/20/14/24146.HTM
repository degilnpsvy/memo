You can pack your initial root filesystem into a compressed cpio archive[2] and pass it to the kernel command line using the [...] or build it as part of the kernel image using the INITRAMFS_SOURCE menu option during kernel configuration. With the latter, you may either provide the filename of a cpio archive or the path name to a directory tree containing your initramfs layout. During boot, the kernel extracts the files into an initramfs root filesystem (also called rootfs) and executes a top-level /init program if it finds one. This method of obtaining an initial rootfs is especially useful for embedded platforms, where all system resources are at a premium. To create an initramfs image, use mkinitramfs. Look at Documentation/filesystems/ramfs-rootfs-initramfs.txt for more documentation.

[2] cpio is a UNIX file archival format. You can download it from www.gnu.org/software/cpio.
