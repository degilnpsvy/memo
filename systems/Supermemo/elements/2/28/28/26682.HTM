LINUX搭建IPv6转发最小环境

发表于 2013 年 6 月 28 日

设备

使用三台机器，一台WINXP笔记本，一台LINUX设备，一台WIN7 PC。
其中，WINXP笔记本连接LINUX设备的eth0口，WIN7 PC连接LINUX设备的eth2口。

详细配置如下图所示：

配置

首先：对WINXP笔记本配置

XP默认不启用IPv6，需要

1
ipv6 install

安装IPv6协议簇，再配置IP地址

1
netsh interface ipv6 add address local 1010::bbbb:10

然后：对WIN7 PC配置

1
netsh interface ipv6 add address local 3030::bbbb:30

LINKLOCAL地址为自动生成不需要配置。

最后：对LINUX设备配置

开启eth0与eth2口：

1
2
[root@localhost code]# ifconfig eth0 up
[root@localhost code]# ifconfig eth2 up

新增GLOBAL IP地址：

1
2
[root@localhost code]# ip address add dev eth0 local 1010::aaaa:1
[root@localhost code]# ip address add dev eth2 local 3030::aaaa:3

查看IP地址，如下红色部分是执行上述两条命令后新增：

此时，从PC PING设备的eth2口的任意IP都是没有问题的。
但是想从PC PING设备的eth0口，就不行了。
给PC添加一条路由：

1
netsh interface ipv6>add route 1010::/64  local 3030::aaaa:3

在PING还是不行，因为设备没有从eth2口到eth0口的路由。
设备上添加路由：

1
2
[root@localhost code]# route -A inet6 add 1020::/64 eth0
[root@localhost code]# route -A inet6 add 3030::/64 eth2
