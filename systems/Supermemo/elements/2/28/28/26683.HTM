怎么写BASH日志接口

发表于 2013 年 6 月 16 日

接口要求

输出目的地

控制台，文件，syslog

附加信息

对于输出的附加信息，最好能包括，文件名，函数名，行号，日志级别，进程PID，时间等信息

接口风格

不同的语言，日志功能类似，接口风格迥异，如
JAVA风格：

1
2
Logger logger = Logger.getLogger(Test. class );
logger.debug( "this is a debug message" );

C++风格：

1
VLOG(1) << "I'm printed when you run the program with --v=1 or higher";

C风格：

1
ldebug("this is a debug message for %d", i++);

这里，我们选择C语言风格，因其简洁。

根据上述要求，我们的SHELL日志接口最终的样子应该是：

1
ldebug "this is a debug message for $i";

实现

我们的SHELL日志接口最终调用之前的C日志接口，将C接口封装成命令行(clogger)调用，所以，有的东西是不需要在SHELL里面实现的，比如输出目的地，附加信息里，时间信息不需要，由于调用的是新命令行，启动了新进程，所以，进程PID也变了，为此，需要传人进程PID。

如下信息都必须从SHELL里面传入：
1. 文件名
众所周知，SHELL会将调用命令行通过$0传入，于是，可以通过分析$0获取文件名。

1
CSCRIPT_NAME=$(basename $(realpath $0));

2. 函数名
BUSH提供$FUNCNAME变量来获取函数调用栈，比如可以通过

1
${FUNCNAME[1]}

来获取上一层函数调用名称。

3. 行号

1
$LINENO

4. 日志级别
传入，这里我们支持用户日志EMERG，ALERT，CRIT，ERROR，WARN，NOTICE，INFO，DEBUG；程序员日志DBG1，DBG2，DBG3，DBGV。

5. 进程PID

1
$$

有了上述信息，我们可以写出这样的语句：

1
2
echo $(date +"%T") DEBUG $$ $CSCRIPT_NAME "($FUNCNAME:$LINENO) | what you want to say"
20:06:49 DEBUG 38024 filename (funcname:22) | what you want to say

样子上看，差不多了，但是，我们要提供的是一个接口而不是每次都要输入这一大串前缀，所以，最好先将其封装成函数，例如，封装成名为lbase的函数，然后将要打印的参数传入，此时$FUNCNAME就应该是上层函数调用名 ${FUNCNAME[1]}，而$LINENO没有提供调用栈风格的${LINENO[1]}，所以，得从调用方传入，为了最大限度重复利用代码，日志级别也从调用方传入。

这样，函数调用就变成了：

1
lbase DEBUG $LINENO "what you want to say"

一下子简洁了许多，看看我们的lbase是如何实现的。

1
2
3
4
5
6
7
lbase()
{
    arg_pri=$1;
    arg_lineno=$2;
    shift;shift;
    echo $(date +"%T") $arg_pri $$ $CSCRIPT_NAME "(${FUNCNAME[1]}:$arg_lineno) | $*";
}

但是，现在调用时，每次都输入lbase DEBUG $LINENO 还是比较冗余，我们的目标是最终只输入ldebug，怎么办？宏？

当然，C语言里使用宏解决问题，SHELL里面也有宏，只是它换了个名叫做alias，我们可以这样定义：

1
alias ldebug='lbase debug $LINENO';

这样，调用方只需要直接ldebug what you want to say就可以了。

注意

如果你定义了alias后发现不能使用，需要查看一下shopt

1
2
root/vtp /src/cur # shopt
expand_aliases on

如果expand_aliases为off，需要开启，使用：shopt -s expand_aliases;

linit()
{
    shopt -s expand_aliases;
    if [ x"$CSCRIPT_NAME" = x"" ]; then
        CSCRIPT_NAME=$(basename $(realpath $0));
        if [ x"$CSCRIPT_NAME" = x"" ]; then
            CSCRIPT_NAME="clogger"
        fi
    fi
    if [ x"$APPNAME" = x"" ]; then
        APPNAME=$CSCRIPT_NAME;
    fi
}
 
lbase()
{
    arg_pri=$1;
    arg_lineno=$2;
    arg_funcname=${FUNCNAME[1]};
    if [ x"$arg_funcname" != x"" ]; then
        arg_funcname="-n $arg_funcname";
    fi
    shift;shift;
    clogger -t "$APPNAME" -p $arg_pri -f "$CSCRIPT_NAME" -l $arg_lineno $arg_funcname -i "$$" "$*";
}
 
alias lemerg='lbase emerg $LINENO';
alias lalert='lbase alert $LINENO';
alias lcrit='lbase crit $LINENO';
alias lerror='lbase error $LINENO';
alias lwarn='lbase warn $LINENO';
alias lnotice='lbase notice $LINENO';
alias linfo='lbase info $LINENO';
alias ldebug='lbase debug $LINENO';
alias dbg1='lbase dbg1 $LINENO';
alias dbg2='lbase dbg2 $LINENO';
alias dbg3='lbase dbg3 $LINENO';
alias dbgv='lbase dbgv $LINENO';
 
linit;
