GDB运行时插入代码修改进程标准输出

发表于 2013 年 9 月 18 日

之前设计了一个日志模块CLOG，有几个接口默认输出到标准输出，用于临时调试，给其命名为程序员日志接口。
程序员日志接口的“日志开关”默认处于关闭状态，目的是给编码人员临时调试用，当版本发布时，因为有动态开关，这些代码也不用删除（方便今后还要打日志）。

日志开关

有编译时开关，即，代码可以在编译时就被移除掉了。通常由宏定义实现。
运行时开关，代码在打日志时，通过检查变量值，决定是否输出日志。
最常用运行时开关的当然是命令行参数和读取配置。
命令行参数：
好比出身之前就已指腹为婚，不管在别人肚子里的长得是否漂亮是男是女是否是人都这么定了，有生之年已成定局，从名义上怕是改不了。
读取配置：
相对灵活，启动时读取好比你出生大富人家，DD没长长就养了童养媳。当然，媳妇这东西是可以换的，只是换起来比较伤元气，特别是现代社会法律规定，换一次媳妇就要掉一半血，所以，读取配置虽然可行，但是没多少人会实时读取。

如此说来，这两种方法实时性都不好，对启动了的进程，如果想修改日志级别，那好比人生走了一半，发现日子过的有点惨淡想重新来过，那只能先跳楼再投胎。

有没有办法不用重新投胎即可焕发第二春？

当然可以，比如传统方法，使用信号通知进程，例如rsyslogd可以使用kill –L HUP重新来过，这方法不错，不过信号就像为数不多的茅坑，你占了绝无第二人共用的道理，每人占一个，后来人只有看着憋着的份。
再比如监听文件系统监听端口，这方法也不错，不过为了切个小JJ却要准备一把牛刀实在浪费。

代码注入

我这里介绍一种方法，此方法可以在不停止运行进程的情况下，不修改编译现有代码的情况下，直接修改变量值，程序员话称，动态插入可执行代码，如果你熟悉WINDOWS，代码注入总不会陌生，在LINUX下，GDB不止是一调试器，它也是一款不错的动态插入代码工具。

GDB里面，有个强大的CALL功能，使用CALL，你可以调用加载到进程空间的所有函数。
使用CALL+TAB查看可调用的函数，比如，CLOG日志库里面，这些函数是以Set开头的函数，有如下：

#!/bin/bash
#
# 1. modify clog's log switch when process is running
# 2. redirect stdout/stderr to file
# @notice haven't test multi-thread and multi-process.
 
lerror()
{
echo "$@" 1>&2;
}
 
ltrace()
{
echo "$@" > /dev/null
}
 
ldebug()
{
echo "$@"
}
 
usage()
{
ldebug "usage:";
ldebug "    $0 PID/PROCESSNAME STDOUT-PATH [LOG-LEVEL]";
ldebug "    LOG-LEVEL - dbg1(1), dbg2(2), dbg3(3), dbgv(4). default is 4."
ldebug "e.g:";
ldebug "    $0 a.out /log/redirect.log 2";
ldebug "    $0 29219 /log/redirect.log";
}
 
pid_running()
{
    arg_pid=$1;
 
    if [ "$arg_pid" == "" ];then
        return 1;
    else
        find_pid=`ps -p $arg_pid | grep $arg_pid | awk '{print $1}'`
        if [ "$arg_pid" = "$find_pid" ]; then
         return 0;
        fi
    fi
 
    return 1;
}
 
attach()
{
arg_pid=$1;
arg_path_stdout=$2;
arg_log_switch=$3;
arg_path_stderr="";
pid_running $arg_pid;
if [ $? -ne 0 ]; then
lerror "process $arg_pid is not running";
return 10;
fi
if [ "$arg_log_switch" = "" ]; then
arg_log_switch=4;
fi
 
touch $arg_path_stdout >/dev/null 2>&1;
if [ $? -ne 0 ];then
lerror "can't touch redirect file $arg_path_stdout";
return 11;
fi
tmp_gdb_file="/tmp/redirect.$arg_pid.gdb";
if [ "$arg_path_stderr" = "" ]; then
cat >$tmp_gdb_file <<EOF
call CL_SetLogDynamicSwitchLessThan($arg_log_switch,1)
call $my_tmpfileno = open("$arg_path_stdout",0x109)
call lseek($my_tmpfileno,0,2)
call dup2($my_tmpfileno,1)
call dup2(1,2)
call close($my_tmpfileno)
detach
q
EOF
    else
    touch $arg_path_stderr;
    if [ $? -ne 0 ];then
lerror "can't touch redirect file $arg_path_stdout";
return 12;
fi
cat >$tmp_gdb_file <<EOF
call CL_SetLogDynamicSwitchLessThan($arg_log_switch,1)
call $my_tmpfileno = open("$arg_path_stdout",0x109)
call lseek($my_tmpfileno,0,2)
call dup2($my_tmpfileno,1)
call close($my_tmpfileno)
call $my_tmpfileno = open("$arg_path_stderr",0x109)
call lseek($my_tmpfileno,0,2)
call dup2($my_tmpfileno,2)
call close($my_tmpfileno)
detach
q
EOF
fi
 
ltrace "gdb -p $arg_pid -x $tmp_gdb_file";
gdb -p $arg_pid -x $tmp_gdb_file >/dev/null 2>&1;
lasterror=$?;
rm -f $tmp_gdb_file >/dev/null 2>&1;
if [ $lasterror -ne 0 ];then
lerror "can't redirect stdout/stderr for $1, error($lasterror)";
return 13;
fi
 
ldebug "Redirect process $arg_pid stdout/stderr success";
return 0;
}
 
main()
{
arg_pid=$1;
arg_path_stdout=$2;
arg_path_stderr=$3;
if [ "$arg_pid" = "" -o "$arg_path_stdout" = "" ]; then
usage;
return 1;
fi
expr $arg_pid "+" 10 >/dev/null 2>&1;
if [ $? -ne 0 ];then
arg_pid=`pidof $arg_pid`;
if [ "$arg_pid" = "" ]; then
lerror "can't find runtime process $1";
return 2;
fi
echo $arg_pid | grep " " >/dev/null 2>&1;
if [ $? -eq 0 ];then
lerror "I don't know how to choose pids from $arg_pid for process $1";
return 3;
fi
ltrace "parpare to modify runtime process $1($arg_pid)'s log level!";
else
ltrace "parpare to modify runtime process $arg_pid's log level!";
fi
attach $arg_pid $arg_path_stdout $arg_path_stderr;
return $?;
}
 
main $*
exit $?
