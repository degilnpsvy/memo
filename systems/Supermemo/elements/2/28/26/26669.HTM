GLOB

"Glob" is the common name for a set of Bash features that match or expand specific types of patterns.

* 匹配0或多个字符(不能匹配".")
? 匹配1个字符
[abc] 匹配括号里的任意一个字符
{b*,c*,*est*} 匹配括号里的任意一个串，例如
> ls {[0-9]*,[a-z]*}.{log,sh}
ls: cannot access [0-9]*.sh: No such file or directory
ls: cannot access [a-z]*.log: No such file or directory
324.log 3aa.log a.sh*
预定义 [:alnum:] [:alpha:] [:blank:] [:cntrl:]
[:digit:] [:graph:] [:lower:] [:print:]
[:punct:] [:space:] [:upper:] [:xdigit:]，
例如，匹配数字或"_"或"." [[:digit:]_.]
^或! 不匹配字符，例如，匹配非空白字符[![:space:]]

文件

-b FILE 块设备 例如 test -b /dev/sda1 && echo true
-c FILE 字符设备 例如 test -c /dev/stdin && echo true
-d FILE 目录 例如 test -d / && echo true
-e FILE 不管是啥，存在即可 例如 test -e / && echo true
-f FILE 常规文件 例如 test -f a.txt && echo true
-h/-L FILE 软链接 例如 test -L /dev/stdin && echo true
-s FILE 文件大小大于0 例如 rm -f a.txt; touch a.txt; test -s a.txt || echo false
-x FILE 具有可执行权限 例如 test -x /bin/sh && echo true

除了-h/-L外，其他的检查都会先dereference symbolic links。

数组

SHELL语法 类似C++ 语法 解释
arr=(aa bb 23) const char* arr={“aa”, “bb”, “23”}; 声明并初始化
${arr[@]} arr 获取整个数组
${#arr[@]} arr.length() 长度
${arr[i]} arr.at(i) 获取下标位置i处值
arr[i]=value arr[i] = value 设置下标位置i处值
unset(arr[i]) arr[i] = NULL 置空下标位置i处值
${arr[@]:offset:length} arr.sub(offset-1, length) 截取子数组，从下标为offset-1开始，长度为length的

arr=("${arr[@]:1:2}") arr = arr.sub(1, 2) 赋值子数组，注意语法，如果不加括号，会当成字符串赋值（且不能自己赋值给自己），如果不加引号，会按当前IFS重新划分数组

重定向问题

strace -p 75750 > a.txt 2>&1
strace -p 75750 2>&1 > a.txt
上面两行哪个对？
第一行对，首先把1>a.txt，然后把2>1，所以2也是1的a.txt
第二行错，首先把2>&1，这时候2被赋值为1的STDOUT，然后把1赋值为a.txt，完全不能重定向STDERR到a.txt
