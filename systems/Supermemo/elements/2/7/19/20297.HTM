Introduction to Initialization

To begin using SDL in your program SDL_Init() must be called before most other SDL functions. The role of SDL_Init() is to properly initialize the SDL library and start each of the various subsystems requested as part of the call.

  •  The Event Handling, File I/O, and Threading subsystems are initialized by default. To initialize other subsystems you must specifically call them. Multiple subsystems may be OR'd together.

  • Example:

    • SDL_Init(SDL_INIT_VIDEO|SDL_INIT_AUDIO);

    • This initializes the 3 default subsystems plus the Video and Audio subsystems.

It should be noted that on some operating systems, SDL_Init() will fail if SDL_main() has not been defined as the entry point for the program. Calling SDL_SetMainReady() prior to SDL_Init() will circumvent this failure condition, however, users should be careful when calling SDL_SetMainReady() as improper initalization may cause crashes and hard to diagnose problems.
