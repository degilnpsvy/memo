Figure 3.5 has the following fields:

l       Config1(M)

Continuation bit, 1 if Config2 is implemented.

l       Config1(MMUSize)

The size of the TLB array (the array has MMUSize+1 entries).

l       L1 I-cache, L1 D-cache

For each cache, this reports three values:

S:Number of cache index positions is 64 × 2S. Multiply by associativity to get the number of lines in the cache.

L:Zero means no cache at all; otherwise, tells you that the cache line size is 2 × 2L bytes.

A:Associativity—this cache is (A + 1)-way set-associative.

So if (IS, IL, IA) is (2, 4, 3), the cache has 256 sets/way, 32 bytes/line, and is four-way set-associative: That’s a 32-Kbyte cache.

l       Config1(C2)

1 if there’s a coprocessor 2 fitted (that would be likely to be some very application-specific coprocessor).

l       Config1(MD)

1 if the old MDMX ASE is implemented in the floating-point unit (see section B.3).

l       Config1(PC)

There is at least one performance counter implemented; see section 12.4.

l       Config1(WR)

Reads 1 if your CPU has at least one watchpoint register; see section 12.2.

l       Config1(CA)

Reads 1 when the MIPS16e compressed-code instruction set is available; see section 12.1.

l       Config1(EP)

Reads 1 if an EJTAG debug unit is provided; see section 12.1.

l       Config1(FP)

A floating-point unit is attached.

l       Config2(M)

Continuation bit, 1 if Config3 is implemented.

l       Config2(TU)

Implementation-specific bits related to L3 cache, if fitted. Might even be writable.

l       Config2(TS,TL,TA)

L3 cache size and shape—encoded just like Config1(IS, IL,IA).

l       Config2(SU)

Implementation-specific bits for secondary cache, if fitted. Can be writable.

l       Config2(SS, SL, SA)

Secondary cache size and shape, encoded like Config1 (IS, IL,IA), above.
