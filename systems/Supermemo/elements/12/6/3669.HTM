Cygmon Command List

The following documentation describes usage of all the commands to type at the Cygmon command prompt. Invoke any command with enough of its command name to specify the command. Some commands have aliases (one letter abbreviations for commands which do not have unique first letters). Aliases for all commands are shown in the help screens. variable text signifies the text that you replace.

baud

A baud speed command sets the baud speed of the active serial port with the speed argument. For instance, baud 9600 sets the speed of the active port to 9600 baud.

break

A break location command displays and sets breakpoints in memory; use zero arguments or one argument, and, with zero arguments, display a list of all currently set breakpoints at location. For instance, break 4ff5 sets a breakpoint at the 4ff5 address.

disassemble

A disassemble location command disassembles the contents of memory. Because of the way breakpoints work, all instructions display and breakpoints are not visible in the disassembled code. The disassemble command takes zero arguments or one argument. When calling with zero arguments, disasemble from the current pc point counter (of the user program). Disassembling begins from the location that you specify. After using a previous call with no arguments, disassembling begins in the next area of memory after the one that you previously disassembled. For instance, disassemble 45667000 displays disassembled code starting at the 45667000 location.

dump

The dump location command shows a region of 16 bytes around a location that you specify, with an alignment to 16 bytes; a dump 65 call shows all bytes from 60 through 6f. A dump 44f5 command displays 16 bytes starting with 44f0 and ending with 44ff.

go

The go location command starts program execution; use zero arguments or one argument. If you provide no argument, go starts execution at the current pc point counter. If you specify a location argument, go sets the pc point counter to that location, and then starts execution at that location. A go 40020000 command, for instance, sets the pc point counter to 40020000, and starts program execution.

help

The help command input without arguments shows a list of all available commands with a short description of each one. With command as an argument, display usage for a specified command and a paragraph describing the command. A help command name command allows for providing names of extensions or arguments (arguments are optional). A help foo command, for instance, displays help for foo.

load

The load command switches the monitor into a state where it takes all input as S-records and stores them in memory. The monitor exits this mode when a termination record is rerached, or certain errors (such as an invalid S-record) cause the load to fail.

memory

The memory.size location value command lets you view and modify single locations in memory. memory can take a size extension, followed by a period and no space and with the number of bits that you specify to view or modify (options are 8, 16, 32, and 64). Without a size extension, the memory command defaults to displaying or changing eight bits at a time. memory can take one or two arguments, independent of a size extension. With one argument, display the contents of the location; with two arguments, replace the contents of location with a value. A memory.8 45f6b2 57 command places the 8 bit value 57 at the 45f6b2 location.

port

The port port number command allows control over the serial port in use by the monitor; port takes zero arguments or one argument. Call port with zero arguments to display the port currently in use by the monitor. Call port with one argument to switch the port in use by the monitor to a number that you specify, in order then to print out a message on the new port confirming the switch. A port 1 command switches the port in use by the monitor to port 1. port is not available for all targets.

register

The register register name value command allows you to view and to manipulate register contents; register can take zero arguments, one argument, or two arguments. When you call register with zero arguments, register displays the values of all registers. When you call register with only a register name argument, register displays the contents of the register that you specify. When you call register with both a register name and a value, register places that value into the register that you specify. A register g1 1f command places the 1f value in the g1 register.

reset

The reset command resets the board.

step

The step command causes one instruction of the user program to execute, then returns control to the monitor. step can take zero arguments or one argument. If you do not provide arguments, step executes one instruction at the current pc point counter. step executes one instruction at a location that you specify. step executes one instruction at a current pc point counter.

terminal

A terminal type command sets the type of the current terminal; with the type argument, vt100 and dumb are the only available terminal types. A line editor uses this information to determine how to update a terminal display. A terminal dumb command sets the type of the current terminal to a dumb terminal.

transfer

A transfer or $ function transfers control to the GDB stub. You do not actually need to call this function, since connecting to the board with GDB will automatically call it. The transfer command takes no arguments. The $ command does not wait for a return, but executes immediately. A telnet setup in line mode will require a return, since the host computer does not pass any characters to the monitor until you press the return key. All debugging stops when you issue this command to disconnect from a board.

unbreak

The unbreak command removes breakpoints from memory; it takes one argument, the location from which to remove the breakpoint. An unbreak 4ff5 command removes a previously set breakpoint at the 4ff5 memory location.

usage

The usage command shows the memory in use for the monitor by category. Despite its name, it has nothing to do with the usage of any other command.

version

The version command displays the version of the monitor.

Cygmon APIs

read and write are the only supported application programming interfaces (APIs).

read

An int read(int fd, char *ptr, int amt); command reads the amount of bytes that you specify (an integer, amt) into ptr from the current serial port, returning an integer. Systems ignore the fd (file descriptor) argument.

write

An int write(int fd, char *ptr, int amt); command writes the bytes of data that you specify (an integer, amt) from ptr to the current serial port, returning an integer. If the program is running in GDB stub mode, the output will appear in GDB. Systems ignore the fd (file descriptor) argument.
