Using Cygmon for Remote Debugging

Cygmon contains a GDB remote stub (sub-routine code running on a remote system exchanging with GDB) with which you can perform full debugging from a host running GDB to a target running Cygmon, the GNUPro ROM monitor. See also http://sources.redhat.com/ecos/docs-latest/redboot/redboot.html for information about RedBoot, which provides a wide set of tools for downloading and executing programs on embedded target systems, as well as tools for manipulating a target system’s environment, for both product development (debug support) and for end product deployment (flash and network booting).

The debugging stub for Cygmon is specific to the architecture of the remote machine; for example, the following remote stubs work with GDB: i386-stub.c (for Intel 386 and compatible architectures), m68k-stub.c (for Motorola 68000 architectures), sh-stub.c (for Hitachi SH architectures), sparc-stub.c (for SPARC architectures), sparcl-stub.c (for Fujitsu SPARClite architectures.

Switching between Cygmon monitor mode and GDB stub mode is transparent, since Cygmon can detect when GDB is communicating with the target and switch into stub mode. When GDB stops communicating with the target normally, it sends a termination packet to let the stub know when to switch to the Cygmon monitor mode.

The command parser is specifically for Cygmon, to provide necessary functionality in limited space.
