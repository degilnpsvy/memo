as, the GNU Assembler

The GNU assembler, as, can be useful as either a compiler pass or a source-level assembler, emitting a relocatable object file from the assembly language source code.

The object file contains the binary code and the debug symbols.

The GNU assembler is really a family of assemblers, with commonality with object file formats, most directives (or pseudo-ops), and the syntax of symbols, constants, expressions and other system-specific issues that developers expect. Its primary function is to assemble the output of the GNU compiler for use by the GNU linker or the GNU debugger.

If you are invoking the assembler using the GNU compiler, use the -Wa option to pass arguments through to the assembler. Usually you do not need to use the -Wa mechanism, since many compiler command line options are automatically passed to the assembler by the compiler.

The following example’s input emits a standard output with high-level and assembly source on a file.c file.

gcc -c -g -O -Wa,-alh,-L file.c

For more information, see Using as in GNUPro Auxiliary Development Tools.
