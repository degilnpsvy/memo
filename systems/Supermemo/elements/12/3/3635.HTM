cpp, the GNU Preprocessor

cpp is a C-compatible macro preprocessor, working with the GNU compiler, to direct the parsing of C preprocessor directives. Preprocessing directives are the lines in your program that start with a # directive name (a # sign followed by an identifier). cpp merges #include files, for instance, then expands macro definitions, and processes #ifdef sections; another example is #define, a directive that defines a macro (#define must be followed by a macro name and the macro’s intended expansion).

To see the output of cpp, invoke gcc with the -E option, and the preprocessed file will print on stdout. The C preprocessor provides the following four separate facilities.

n Inclusion of header files

Declarations that can be substituted into your program.

n Macro expansion

When defining macros, which are abbreviations for arbitrary fragments of C code, the C preprocessor will replace them with their definitions throughout a program.

n Conditional compilation

Using special preprocessing directives, include or exclude parts of a program according to various conditions.

n Line control

Using a program to combine or rearrange source files into an intermediate file, which is then compiled, use line control to provide a source line’s origin.

There are two convenient options to assemble handwritten files that require C-style preprocessing. Both options depend on using the compiler driver program, gcc, instead of directly calling the assembler.

n Name the source file using the extension, .S (capitalized), rather than .s (assembly language requiring C-style preprocessing).

n Specify a source language explicitly for a situation, using the

-xassembler-with-cpp option.
