Developing with GNUPro Toolkit

Using GNUPro Toolkit for development typically involves the following process.

1. Create a project and edit its source code.

See “The C Run-time Environment (crt0)” on page 40 for more information.

2. Compile your source code with the GNU compiler into intermediate object files.

See “crt0, the Main Startup File” on page 41 for more information.

3. Link object files together with the GNU linker to produce an executable file.

See “The Linker Script” on page 44, for more information.

4. Debug the executable file to find any problems.

See “Using Cygmon for Remote Debugging” on page 57, for more information.

5. Build the executable file.

Using GNUPro Toolkit for development with cross-configurations requires setting up a target from a host system, a process known as embedded development; for a summarization, see “What Embedded Development Means” on page 38. To work with the tools without use of a terminal shell’s command line, see“Using Cygwin” on page 47 and “Using the GNUPro Toolkit Graphical User Interface” on page 63.
