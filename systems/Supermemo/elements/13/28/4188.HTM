13.2 Layering in the Kernel

From one point of view, the kernel is a set of subroutines called from exception handler. The raw post-exception “exception mode” environment on a MIPS CPU is all-powerful and very low-overhead but tricky to program. So with each entry to the kernel you get something like a foreshortened bootstrap process, as each “layer” constructs the environment necessary for the next one. Moreover, as you exit from the kernel you pass through the same layers again, in reverse order, passing briefly through exception mode again before the final eret which returns you to userland.
