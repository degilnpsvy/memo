Recursive merges

A simple criss-cross merge example is shown in Figure 9-8.

 

Figure 9-8. Simple criss-cross merge

The nodes a and b are both merge bases for a merge between A and B. Either one could be used as the merge base and yield reasonable results. In this case, the recursive strategy would merge a and b into a temporary merge base, using that as the merge base for A and B.

Because a and b could have the same problem, merging them could require another merge of still older commits. That is why this algorithm is called recursive.
