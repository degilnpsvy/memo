8.12 Domain-Specific Languages

The goal of metaprogramming in Ruby is often the creation of domain-specific languages,

or DSLs. A DSL is just an extension of Ruby’s syntax (with methods that look

like keywords) or API that allows you to solve a problem or represent data more naturally

than you could otherwise. For our examples, we’ll take the problem domain to

be the output of XML formatted data, and we’ll define two DSLs—one very simple and

one more clever—to tackle this problem.( * For a fully realized solution to this problem, see Jim Weirich’s Builder API at http://builder.rubyforge.org.)
