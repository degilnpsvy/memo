3.6. Designed for portability

All of newlib's functionality builds on a set of seventeen stub functions that newlib uses to hook into the host's execution environment. By modifying this integration layer, you can adapt newlib to just about any system imaginable, from one with no operating system at all, to one based on an embedded RTOS, to one with a complete POSIX operating system.

Newlib's documentation provides details on which stubs are needed for each library function, and you only need to provide stubs for the portions of newlib that you intend to use. For example, newlib has a stub called _fork, but you don't need to do anything with it unless you intend to use system() or fork().
