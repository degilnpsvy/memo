5.4.3 Function return stack frame

The format of the function return stack frame is shown in Figure 5. The stack frame must be 8 byte aligned. However unlike exception stack frames it is the software's responsibility to make sure the stack pointer is aligned before executing a BLXNS instruction that might result in the creation of the stack frame. NOTE: This requirement is already fulfilled by the existing ABI which mandates an 8 bytes alignment on function boundaries.

                                                   SP Offset
Original SP, 8 byte aligned   ->                
                                   Partial xPSR      0x04
New SP, 8 byte aligned        ->  ReturnAddress      0x00
Figure 5 Function return stack frame format

The partial xPSR value in the function return stack frame contains IPSR (bits[8:0]) and SFPA (bit[20]).
