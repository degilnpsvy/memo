4.8.7 NSACR

Non-Secure Access Control Register, address 0xE000ED8C, Resets to UNKNOWN.

CPn Each CPn bit corresponds to one of the coprocessors, and is enumerated as follows
    0 Otherwise valid accesses to the coprocessor from the non-secure state result in a NOCP UsageFault being raised that targets the secure state.
    1 Access to the coprocessor is allowed from any security state.
    NOTE: The value of CP11 is ignored. If the value of this bit is not programmed to the same value as the CP10 field, then the value is UNKNOWN
    CPn bits corresponding to coprocessors that aren't implemented behave as RAZ/WI.
    CPn bits corresponding to coprocessors that are hard wired to only be enabled for a single security domain behave as RAZ/WI.

This register is only accessible from the secure privileged state, accesses from unprivileged states generate a BusFault and access from the privileged non-secure state behave as RAZ/WI.
