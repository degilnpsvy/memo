The SAU configuration defines an IMPLEMENTATION DEFINED number of memory regions. The number of regions is indicated by SAU_TYPE.SREGION,
IPPLK The memory regions defined by the SAU configuration are referred to as SAU_REGIONn, where n is a number from 0 - (SAU_TYPE.SREGION-1).
RRVFP The SAU region configuration fields can only be accessed indirectly using the window registers shown in the following table.

SAU_REGIONn.Enable                  SAU_RLAR.ENABLE
SAU_REGIONn.NSC                     SAU_RLAR.NSC
