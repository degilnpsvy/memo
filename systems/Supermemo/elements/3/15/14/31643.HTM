B5.5.2 Multi-copy atomicity

IBCHK In a multiprocessing environment, writes to memory are multi-copy atomic if all of the following are true:
• All writes to the same location are observed in the same order by all observers, although some of the observers might not observe all of the writes.
• A read of a location does not return the value of a write to that location until all observers have observed that write.

RGJGP Writes to Normal memory are not required to be multi-copy atomic.

RLBGB Writes to Device memory with the Gathering attribute are not required to be multi-copy atomic.

RWHJR Writes to Device memory with the non-Gathering attribute that are single-copy atomic are also multi-copy atomic.
