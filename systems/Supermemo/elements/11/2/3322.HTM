8.7.12 Limited User-Mode Access to “Under the Hood” Features

rdhwr (read hardware register): Allows a user-privilege program to read a few

snippets of information that would normally only be visible to a kernel. See

section 8.5.12.

synci: User-privilege instruction. It specifies an address like any load/store,

and it affects the cache-line-sized, aligned piece of memory that holds the

addressed byte. Use it when writing instructions for subsequent execution.

synci does what is necessary to ensure that the CPU will correctly execute

any instructions that the program previously wrote in this chunk of memory

(often that means a write-back from the L1 D-cache and an invalidate of any

previously held I-cache contents).

sync: Load/store barrier, listed here to emphasize that it’s not just a kernel privileged

instruction. It’s described in section 8.5.9.
