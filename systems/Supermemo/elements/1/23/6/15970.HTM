Besides global memory, threads also share a number of other attributes (i.e., these attributes are global to a process, rather than specific to a thread). These attributes include the following:

l  process ID and parent process ID;

l  process group ID and session ID;

l  controlling terminal;

l  process credentials (user and group IDs);

l  open file descriptors;

l  record locks created using fcntl();

l  signal dispositions;

l  file system–related information: umask, current working directory, and root directory;

l  interval timers (setitimer()) and POSIX timers (timer_create());

l  System V semaphore undo (semadj) values (Section 47.8);

l  resource limits;

l  CPU time consumed (as returned by times());

l  resources consumed (as returned by getrusage()); and

l  nice value (set by setpriority() and nice()).

Among the attributes that are distinct for each thread are the following:

l  thread ID (Section 29.5);

l  signal mask;

l  thread-specific data (Section 31.3);

l  alternate signal stack (sigaltstack());

l  the errno variable;

l  floating-point environment (see fenv(3));

l  realtime scheduling policy and priority (Sections 35.2 and 35.3);

l  CPU affinity (Linux-specific, described in Section 35.4);

l  capabilities (Linux-specific, described in Chapter 39); and

l  stack (local variables and function call linkage information).
