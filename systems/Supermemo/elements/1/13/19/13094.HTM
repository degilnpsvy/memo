11.1. Motivation

Most modern operating systems are multitasking, meaning that they can run many programs "at the same time". In reality, a computer can only run one process per processor at a given momemt, but more processes can be in a running state by sharing the CPU is a round-robin fashion.

Most such systems save memory using shared objects. A shared object is typically a subprogram that is used by more than one program. If two programs are running at the same time, and they use some of the same subprograms, then the shared subprograms need only be loaded into memory once, provided that the subprograms are reentrant.
