4.9. Binary/Octal/Hexadecimal Conversions

If a = bn, then each digit in a base a number will be n digits in the base b number.

Since 16 = 24, each hexadecimal digit corresponds to 4 bits.

Since 8 = 23, each octal digit corresponds to 3 bits.

FFC3.216 = 1111 1111 1100 0011 . 00102

When converting from binary to base 2N, always group bits beginning at the radix point and move outward.

11001.112 = 011 001 . 1102 = 31.68

111010.0112 = 0011 1010 . 01102 = 3A.616

When converting large numbers from decimal to binary, it is often easier to first convert to hexadecimal, and then from hexadecimal to binary. This greatly reduces the number of steps in the conversion from decimal.
