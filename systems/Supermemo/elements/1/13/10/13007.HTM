5. Conclusion and References

The tutorial and description of advanced features has provided an introduction to all of the functionality available in Check. Hopefully, this is enough to get you started writing unit tests with Check. All the rest is simply application of what has been learned so far with repeated application of the “test a little, code a little” strategy.

For further reference, see Kent Beck, “Test-Driven Development: By Example”, 1st ed., Addison-Wesley, 2003. ISBN 0-321-14653-0.

If you know of other authoritative references to unit testing and test-driven development, please send us a patch to this manual.
