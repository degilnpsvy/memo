7.7. Code Size

Make it a life-long career goal to eliminate waste in your code. Try to make every program you write simpler than the last one. Examine the code you've written and think about how you could have made it simpler and more general-purpose.

Smaller programs are usually better programs. Programs that are longer than necessary cost more to write and maintain, and usually run slower. Always strive to write the shortest, simplest program possible. Adding code to improve functionality is acceptable, although there are also many situations where functionality can be added by modifying existing code without making it longer. Adding redundant or useless code is wasteful, but unfortunately common.

Many programmers think it's "cool" to use the most sophisticated approach to solve a problem. In reality, it's just more expensive and limited. Don't let your ego ruin your code. If you want to show the world how clever you are, do so by demonstrating your ability to understand the heart of a problem and find a simple, elegant, efficient, and generalized solution that will require minimal maintenance.

Make sure your code is completely factored (contains no redundancy), and make an effort to optimize for both size (# of lines) and speed (execution path).
