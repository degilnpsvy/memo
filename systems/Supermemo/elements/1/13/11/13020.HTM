2.1. Motivation

Solid knowledge of the tools used in programming is critical to productivity. A programmer who doesn't know how to use the operating system is like a carpenter who doesn't know how to use a saw.

A programmer who is adept with the available tools can literally be many times more productive than one who isn't. Knowing the tools well not only saves time when using them, it eliminates distractions from programming that might require you to restart your thought process.

The key to productive programming is momentum. Develop your skills with programming tools so that they become second-nature. This will allow you to stay focused on the code you are developing, and keep your train of thought. A good programmer with a solid plan is often limited only by how fast [s]he can type.

Students should take every possible opportunity to learn more about the operating systems in use today, especially Unix, Mac OS X (which is based on Unix) and Windows. There are no courses available that cover operating system use to the depth that programmers will need. This knowledge must be acquired through self-study, by reading books and online material, and by trial and error.
