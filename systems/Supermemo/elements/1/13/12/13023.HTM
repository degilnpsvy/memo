2.4. Unix Basics

This is a very brief introduction to Unix. Consult a Unix textbook and online documentation (man command) for more details.

2.4.1. Unix Files

A file is simply a sequence of characters (letters, digits, punctuation symbols, and control characters) stored on a disk and given a unique name. You have likely already used files to store documents, spread sheets, and emails. Virtually all information stored on disks is stored in files, including programs, input and output data.

Different operating systems use different file formats. Unix text files terminate each line with the line feed (also called newline) character (ASCII/ISO character 10). DOS/Windows uses both a carriage return (ASCII/ISO character 13) and a line feed. Older Mac programs use only a carriage return, and some Mac-only editors on OS X may do the same, although Mac OS X is Unix, and most Mac tools now use the Unix format.

To compound the problem, many Unix editors and tools also run on Windows and MAC OS X (which is Unix-based). As a result, many files end up with a mix of line-terminations after being edited on multiple platforms.

This situation can cause issues with some programs that process text files (e.g. assemblers, preprocessors, compilers).

The dos2unix and unix2dos commands convert text files between the DOS and Unix standards. If you've edited a file on a non-Unix system, and are now using it on a Unix system, you can clean it up by running:

	    dos2unix filename

2.4.2. Unix Filesystem

A filesystem is a way of organizing all the files stored on a disk. Disks are organized in much the same way as an ordinary file cabinet. A disk may be divided into partitions, which are analogous to drawers in a file cabinet. Within each partition are directories, which are analogous to folders in a drawer, and in fact are often called folders. Each directory/folder contains files, which are analogous to the papers in a folder. Directories/folders may contain other directories/folders to add another level of organization.

A Unix filesystem can be visualized as a tree, with each file and directory contained within another directory. The only file or directory that is not contained in another is the root directory. There is exactly one root directory on every Unix system.

Figure 2.1, “Sample of a Unix Filesystem” shows a small portion of one possible Unix filesystem. On a real Unix system, there are usually thousands of files and directories.

The full pathname of a file or directory denotes the path from / to the file or directory. For example, the pathname of Sue's .cshrc file is /home/sue/.cshrc, and the pathname of the ape command is /usr/local/bin/ape.

Unix filesystem trees are fairly standardized, but most have some variation. For instance, all Unix systems have a /bin and a /usr/bin, but not all of them have /home or /usr/local.

The root directory or folder (called /) is the ancestor of all directories, and the parent directory of bin and home. The bin and home directories are subdirectories (children) of /. Likewise, joe and sue are subdirectories of /home, and grandchildren of /. All of the files in and under home comprise a subtree of home.

Each user has a home directory, which may be under different parent directories on different Unix systems. In the example above, /home/joe is the home directory for user joe, and /home/sue is the home directory for user sue. This is the conventional location for home directories on FreeBSD and Linux, which are two specific types of Unix. On a Mac OS X system, which is another brand of Unix, Joe's home directory would be /Users/joe instead of /home/joe. Most of the files owned by ordinary users are in their home directory, or its subdirectories.

2.4.3. Processes and the Current Working Directory

A process is a running program.

The current working directory is a property of every Unix process (running program).

It can be referred to as a single period ('.').

The parent directory of the current directory can be referred to as '..'.

Your home directory can be referred to as "~/".

2.4.4. Pathnames

A pathname is a specification of a file or directory showing at least some of its ancestors.

A full pathname, also known as absolute pathname starts at the root directory, with '/'. For example:

	    /home/CS/cs315/601/stud4391

A relative pathname starts at the current working directory '.' or an ancestor. I.e., it is relative to a starting point other than the root directory.

	    ./Prog1/prog1.mal

or just

	    Prog1/prog1.mal

or assuming Prog1 is a child of your home directory

	    ~/Prog/prog1.mal

The relative path of /home/CS/cs315/pub from a home directory of a cs315 user would be

	    ../../pub
