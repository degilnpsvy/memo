2.2. Logging In

The computers in most EMS labs are dual-booted. While the machine is in BIOS POST (showing BIOS messages in the initial boot phase), pressing F12 will invoke a boot menu. Selecting boot from network will boot a Linux image over the network, which will lead to a chooser screen where you can select miller or weise. Selecting the internal hard disk will boot Windows.

Caution

Because of current network issues, the computers in EMS E270 cannot be booted from the network using F12. As a temporary workaround, boot into Windows and run the Cygwin-X application:

Start -> Programs -> Computer Science Applications -> Communication -> X-Win weise.

The X-Win miller link is also currently not working, but you can run "ssh miller" from a terminal on weise if you need to log into miller. ( e.g. to change your password, which should not be done on weise. )

Whether logging into miller/weise from the computers on campus, or from your laptop or desktop machine at home, you will be using the Unix command-line interface (CLI) for work in this course.

In addition to running Unix commands from the command-line, you will be using some text-based software (such as text editors) that require the ability to manipulate an ASCII terminal, i.e. move the cursor around, highlight or change the color of text, etc. To accomplish this, the software needs to know what type of terminal or terminal emulator you are using. Each type of terminal may use different "magic sequences" (sequences of characters sent to the terminal to make it perform a function such as clearing the screen, moving the cursor, etc.).

Most of the terminal emulators in use today, including the Terminal windows used in the Java Desktop in E270, the Mac Terminal application, and PuTTY, are based on the "xterm" terminal.

If the system asks for a terminal type by displaying a string such as

	TERM = (unknown)

type "xterm" (without the quotes), unless you know that your terminal emulator emulates a different type.

Caution

Do not use "xterm-color" on miller and weise. It is not recognized by the OS installation, and some programs will not function.

When logging in from the lab machines in E270, be sure to select the Java Desktop session type. This may or may not be the default, depending on what you used before. The session type is selected after selecting a machine to log into (e.g. miller or weise), but before actually logging in.
