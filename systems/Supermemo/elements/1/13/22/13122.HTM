15.3. Traps

Exceptions are caused by internal conditions in the CPU, most often errors caused by faulty programs.

  • Arithmetic overflow occurs when the absolute value of a result is too large for the format.
  • Arithmetic underflow occurs when the absolute value of a result is too small for the format. This occurs only in floating point formats, or other formats that can represent values between 0 and 1.
  • The divide by 0 trap is self-explanatory.
  • Illegal instruction traps occur when the word read from the address in the program counter is not a valid instruction. This usually means the program counter was somehow corrupted during program execution. It could also be cause by faulty memory where an instruction was stored.
  • A bus error occurs when there is an attempt to access some resource on the bus that does not exist, or cannot be accessed in the way requested. For instance, it could be an attempt to access a memory address beyond the amount of memory installed.
  • A segmentation fault is an attempt to access memory in an illegal way. Modern processors allow the operating system to tag segments of memory as code or data, for example. Attempts by a program to modify memory in the code segment will result in a segmentation fault.
